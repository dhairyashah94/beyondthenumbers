Publish Once. Update Forever With One Click.

A one‑time, no‑coding setup that connects your site to GitHub and Netlify. After this, you edit a file in your browser and the live site updates by itself in about 30 seconds — no more dragging folders.

⏱ 15–20 min, once 🧩 No coding 💻 In your browser 🆓 Free
Phase 1 — Put your files on GitHub

GitHub is the filing cabinet that holds your website.

Think of GitHub as cloud storage built for website files. You upload your folder here once; later you'll edit files right on this site and they'll auto‑publish. Have your unzipped website folder ready on your computer.

1

Create a free GitHub account

Go to github.com/signup. Enter your email, pick a username and password, and verify your email. It's free — no card needed.

What you'll seeOnce signed in, a mostly‑empty dashboard with a green button to make something new. That's next.
2

Create a new repository

A "repository" (or "repo") is just a project folder. Click the + icon at the very top‑right of the page, then choose New repository.

In the Repository name box, type behind-the-numbers (no spaces). Leave everything else as it is. Make sure Public is selected. Then scroll down and click the green Create repository button.

Why public?It's needed for the free website hosting and only exposes your site's files — which are public on the web anyway.
3

Upload your website files

On the new repo page, find the line that says "uploading an existing file" and click that link. (If you don't see it, click the Add file button → Upload files.)

Open your unzipped website folder on your computer, select all the files inside it — index.html, article.html, spacex-ipo-valuation.html, styles.css, site.js, and the image files — and drag them onto the upload area.

Important: drag the files themselves, not the outer folder. index.html must sit at the top level.

4

Save them (this is called a "commit")

Scroll down to the box that says Commit changes. You can leave the default message ("Add files") or type something like "First upload." Then click the green Commit changes button.

What you'll seeThe page reloads and now lists all your files. That's it — your whole site is safely on GitHub. ✅
Phase 2 — Connect Netlify so it goes live

Netlify turns your GitHub files into a real website.

Netlify watches your GitHub repo. Every time you change a file there, Netlify automatically rebuilds your live site. You connect them once here.

5

Sign up for Netlify with your GitHub account

Go to app.netlify.com/signup and click GitHub to sign up. Using GitHub to sign in is what links the two together. Approve the access it asks for.

6

Import your repository

On your Netlify dashboard click Add new site → Import an existing project. Choose Deploy with GitHub. If asked, authorize Netlify to see your repositories.

From the list, click your behind-the-numbers repo.

7

Deploy

A settings screen appears. You don't need to change anything — leave every box empty/default. Scroll down and click the Deploy button.

Wait about 30–60 seconds while it builds. When it's done you'll see a status of Published and a link near the top like random-name-123.netlify.app. Click it — that's your live site. 🎉

8

Give it a nicer name (optional)

To change the random address, go to Site configuration → Change site name, and type something like behind-the-numbers. Your link becomes behind-the-numbers.netlify.app — that's the link you post on LinkedIn and share with friends.

From now on

How you publish a new brief — the whole routine

No folders, no Netlify, no dragging. Just edit the file on GitHub:

A
Go to your repo on github.com and click index.html to open it.
B
Click the pencil ✏️ icon (top‑right of the file) to edit it.
C
Find a Daily Brief block, copy it, paste it at the top of the list, and change the date and text. (I leave comments in the file marking exactly where.)
D
Click Commit changes. Netlify republishes automatically within ~30 seconds. Done.

Easier still: paste your notes to me, I rewrite the brief in your voice and hand you the updated file or the exact text to drop in.

If something looks off
My site link shows a list of files, not the website.
Your index.html got uploaded inside a sub‑folder. Re‑upload so index.html sits at the top level of the repo (not inside another folder).
I edited a file but the live site didn't change.
Give it a minute, then refresh with the cache cleared (Ctrl/Cmd+Shift+R). On Netlify, the Deploys tab shows whether the latest build finished — a green "Published" means it's live.
Do I ever need the "drag the folder" method again?
No. That was the old way. Once GitHub and Netlify are linked, editing on GitHub is all you need. The two methods do the same job — pick one.
Will headlines, the date, and market data still auto‑update?
Yes — those refresh on their own for every visitor regardless of how the site is hosted. This setup is only about making your edits easier to publish.
Is any of this going to cost money?
No. GitHub and Netlify both have free tiers that comfortably cover a site like this. You'd only pay if you wanted a custom domain (e.g. behindthenumbers.com), which is optional.
